<p>Joshua Bloch is a master at API design. Check out <a href="http://www.infoq.com/presentations/effective-api-design" rel="nofollow">this video</a> from him about API design. Google his name and API to find more blogs that he has written. The main points are to make sure that you write client code that uses your API as you write it to make sure that it feels natural. As you write your API, write a small web interface, cli interface, and desktop app to ensure that whoever uses your code will be able to easily. </p>

<p>In terms of specifics, using several different ways of transporting messages and data back and forth would be best. Try to write it using SOAP/XML and JSON so that the programmers using it can choose what makes sense for them.</p>
