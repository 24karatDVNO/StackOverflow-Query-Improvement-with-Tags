<p><code>jobject</code> could be any java object while <code>jclass</code> is supposed to be an object representing <code>java.lang.Class</code>. In C API, <code>jobject</code> and <code>jclass</code> is the same thing while C++ API is trying to enforce type safety and declares those as different types where <code>jclass</code> inherits <code>jobject</code>. The second argument is supposed to represent a class, so <code>jclass</code> is preferred. Even if in C it doesn't matter, it can hint the developer or even save you time if you ever decide to switch to C++. Read more about JNI types <a href="http://download.oracle.com/javase/1.3/docs/guide/jni/spec/types.doc.html" rel="nofollow">here</a>.</p>
