<p>Well, there are simpler alternatives to SQLite - you should at least consider some kind of ORM for Android. 
It'll let you to persist Java objects in a few lines of code instead of using <code>SELECT</code>, <code>UPDATE</code>, and marshalling data. </p>

<p>Consider <a href="http://www.db4o.com/android" rel="nofollow">db4o</a> or something like that - it's rather large (~1M), though.</p>
