<p>You write the code on the server that free to do whatever it wants. And that includes adding or changing data as needed. Check the <a href="http://tools.ietf.org/html/rfc5023#section-9.2" rel="nofollow">AtomPub protocol section 9.2</a> that explicitly states: </p>

<blockquote>
  <p>Since <strong>the server is free to alter the
  POSTed Entry</strong>, for example, by changing
  the content of the atom:id element,
  returning the Entry can be useful to
  the client, enabling it to correlate
  the client and server views of the new
  Entry.</p>
</blockquote>
