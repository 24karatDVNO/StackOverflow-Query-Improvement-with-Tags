<p>the <em>SIGSEGV</em> at fault addr <em>6c6c65bc</em> indicates an invalid memory reference or segmentation fault (<a href="http://en.wikipedia.org/wiki/SIGSEGV" rel="nofollow">SIGSEGV</a>).<br>
For exceptions from the native code, you must explicit call on C/C++ side the method <code>ExceptionDescribe()</code>. More details you can find here <a href="http://download.oracle.com/javase/6/docs/technotes/guides/jni/spec/functions.html#wp5234" rel="nofollow">Exceptions</a>.</p>
