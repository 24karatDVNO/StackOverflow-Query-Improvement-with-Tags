<p>whats wrong with using <code>link_to</code> and escaping the string? If you are unsure of the strings safety call something like <code>link_to "Test name", h(link)</code> See <a href="http://api.rubyonrails.org/classes/ActionView/Helpers/UrlHelper.html#method-i-link_to" rel="nofollow">http://api.rubyonrails.org/classes/ActionView/Helpers/UrlHelper.html#method-i-link_to</a> for more information.</p>

<p>This string should be escaped even without the call to h. Try reading this blog post on the <a href="http://yehudakatz.com/2010/02/01/safebuffers-and-rails-3-0/" rel="nofollow">XSS protection in Rails</a>.</p>
