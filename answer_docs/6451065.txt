<p>If it was about just incrementation of number, then using a <a href="https://docs.djangoproject.com/en/1.3/ref/models/fields/#autofield" rel="nofollow">AutoField</a> might be enough, but incrementation of a value prior to some other data is not easy to handle on model definiton.</p>

<p>One possible way is overriding <strong>save()</strong> method.</p>

<p><a href="https://docs.djangoproject.com/en/1.3/ref/models/instances/#saving-objects" rel="nofollow">Check here</a> for basic django saving procedure. And <a href="https://docs.djangoproject.com/en/1.3/topics/db/models/#overriding-model-methods" rel="nofollow">here for</a> how to override default save method. You may try to check if related save action is for a new insertion or an update for an existing record, and if it is insertion, you may set a unique number to that record for specified user. </p>
