<p>if you are willing to pull all of your data into memory and work with it in memory, here is a library that will work really well for your use case</p>

<p><a href="http://casperdatasets.googlecode.com" rel="nofollow">http://casperdatasets.googlecode.com</a></p>

<p>it operates effectively like an in-memory table, and allows you to perform searching, filtering, and SORTING on data, all in memory (and in java). it performs very fast for the number of records that you are trying to work with, and you don't need to integrate with a heavy ORM framework. </p>
