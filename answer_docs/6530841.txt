<p>Neither. That's HTTP. Specifically, <a href="http://en.wikipedia.org/wiki/Basic_access_authentication" rel="nofollow">basic auth</a>.</p>

<p>While this can be done in your server side programming language (e.g. <a href="http://php.net/manual/en/features.http-auth.php" rel="nofollow">in PHP</a> and <a href="http://search.cpan.org/perldoc?CGI%3a%3aAuth%3a%3aBasic" rel="nofollow">a Perl module</a>), it is usually handled by your web server. The specifics depend, of course, on which web server you are using. For example, <a href="http://httpd.apache.org/docs/2.2/howto/auth.html" rel="nofollow">here are instructions for Apache HTTPD</a> and <a href="http://www.microsoft.com/technet/prodtechnol/WindowsServer2003/Library/IIS/abbca505-6f63-4267-aac1-1ea89d861eb4.mspx?mfr=true" rel="nofollow">for IIS</a>.</p>
