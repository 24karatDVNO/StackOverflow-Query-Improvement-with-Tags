<p>I think you should use the <a href="http://msdn.microsoft.com/en-us/library/system.windows.markup.xamlreader.aspx" rel="nofollow"><code>XamlReader</code></a> class, then once you have concrete objects you can easily add them wherever parent control you wish.</p>

<p>Use <a href="http://msdn.microsoft.com/en-us/library/system.windows.markup.xamlreader.load.aspx" rel="nofollow"><code>XamlReader.Load</code></a> if you want to load it via a stream (e.g. <a href="http://msdn.microsoft.com/en-us/library/system.io.filestream.aspx" rel="nofollow"><code>FileStream</code></a>), or <a href="http://msdn.microsoft.com/en-us/library/system.windows.markup.xamlreader.parse.aspx" rel="nofollow"><code>XamlReader.Parse</code></a> if they are in available strings.</p>

<p>These methods will return the root of the constructed object tree (which might even be a <code>Path</code> as well).</p>
