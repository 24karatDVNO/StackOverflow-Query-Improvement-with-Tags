<p>Seems like you're, like so many others, in the beginning of reinvent the famous <a href="http://download.oracle.com/javase/6/docs/api/java/beans/PropertyChangeListener.html" rel="nofollow">PropertyChangeListener</a>/<a href="http://download.oracle.com/javase/6/docs/api/java/beans/PropertyChangeEvent.html" rel="nofollow">PropertyChangeEvent</a>/<a href="http://download.oracle.com/javase/6/docs/api/java/beans/PropertyChangeSupport.html" rel="nofollow">PropertyChangeSupport</a> mechanism. So, do yourself a favor, and instead of creating your own event/listener hierarchy, reuse that generic one, which is specifically tailored for beans property changes (and as a consequence all Java objects).</p>
