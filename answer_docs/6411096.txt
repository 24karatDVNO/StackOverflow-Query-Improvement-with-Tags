<p>If your objects follow the Java Beans spec and you don't care how exactly they're represented as XML, <a href="http://download.oracle.com/javase/6/docs/api/java/beans/XMLEncoder.html" rel="nofollow"><code>XMLEncoder</code></a> from the Java standard API can do the job.</p>

<p>If your objects are not "pure" JavaBeans or you need more control over the generated XML,  <a href="http://xstream.codehaus.org/" rel="nofollow">XStream</a> is a popular XML serialization library.</p>
