<p>Yes. Python trusts the developer to make his own choice about this. Whereas you can forbid access to a property in Java and C#, in Python the developer should know that he should not access a property of another class.</p>

<p>You can prefix a property with two underscores to make it harder to access. See <a href="https://secure.wikimedia.org/wikipedia/en/wiki/Name_mangling#Name_mangling_in_Python" rel="nofollow">name mangling</a>.</p>
