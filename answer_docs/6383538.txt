<p>I think that the best approach is migrating from a manual handling of the sqlite3 database to a Core-Data-based one. <a href="http://developer.apple.com/library/ios/#documentation/Cocoa/Conceptual/CoreData/cdProgrammingGuide.html" rel="nofollow">Core Data</a> provides you with a lot of tools for defining and modelling data, including a super easy (when available) <a href="http://developer.apple.com/library/ios/#documentation/Cocoa/Conceptual/CoreDataVersioning/Articles/vmLightweight.html#//apple_ref/doc/uid/TP40008426-SW1" rel="nofollow">lightweight migration</a> process that will incredibly ease the data migration process between various data base versions.</p>

<p>Using a <a href="http://developer.apple.com/library/ios/#documentation/Cocoa/Conceptual/CoreDataVersioning/Articles/vmLightweight.html#//apple_ref/doc/uid/TP40008426-SW1" rel="nofollow">lightweight migration</a> (or the more complex <a href="http://developer.apple.com/library/ios/#documentation/Cocoa/Conceptual/CoreDataVersioning/Articles/vmMigrationProcess.html#//apple_ref/doc/uid/TP40005508-SW1" rel="nofollow">custom manual migration</a>) your problems will be automatically solved.</p>
