<p><code>count</code> is working just fine. It does however return a <code>NSUInteger</code> primitive and not a pointer to a <code>NSObject</code> subclass. The <code>%@</code> string formatter expects a point to an object and logs the <code>NSString</code> returned from that object's <code>-description</code> method. When you pass it a <code>NSUInteger</code> <code>NSLog</code> assumes it to be an object pointer and dutifully tries to send a <code>-description</code> message to memory address 3 which causes that EXEC_BAD_ACCESS.</p>
