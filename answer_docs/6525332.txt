<p>Any valid way for .Net CLR to call your code will do what you want.  However with that said, it is likely to be very low level, very obscure and very difficult to do.
<br>
A more straightforward approach to reusing your Java or whatever code was if there was some sort of public cross language interface for you to exploit in your .Net skeleton.  The obvious answer is a REST or SOAP wrapper around the Java code but it doesn't have to be that.  It could be CORBA or JMS or all sorts of things.</p>
