<p>In the last release <a href="http://www.telerik.com/products/orm.aspx" rel="nofollow">Telerik OpenAccess</a> introduced a flexible type mapping system which allows developers to easily work with custom DB types/UDTs.  It ships with support for Geometric/Geographic data, and it is easy to create your own type converter to tell OpenAccess how to map any custom types you may have. The type converters work for all <a href="http://www.telerik.com/products/orm/supported-databases.aspx" rel="nofollow">DBs OA supports</a>, one of which is Oracle. :)</p>
