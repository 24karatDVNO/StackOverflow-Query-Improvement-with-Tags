<p>Your ordering looks about right, bu I'm not sure if there is a mandatory sequence for creating UML diagrams for a project. You should really be able to dip in an out of UML when it suits you.</p>

<p>If you want to put a structure on how you use UML, you should have a look into the various software modelling processes that are out there. One that I have used quite a bit is the ICONIX process, <a href="http://www.informit.com/articles/article.aspx?p=167902" rel="nofollow">http://www.informit.com/articles/article.aspx?p=167902</a>. This is a light weight process based on use cases.</p>
