<p>Although it is an old question, but for all those who face this issue and end up finding this question for a possible solution. Here it goes...</p>

<p>Initially caching could be turned off by setting the "CacheForContextType" property of the ModelBuilder to ‘false’ in the OnModelCreating method. This method is defined in DBContext as virtual and needs to be overridden. But in EF 4.1 this property has been removed, since model creation is an expensive process and the Microsoft team wanted to promote a better pattern. <a href="http://social.msdn.microsoft.com/Forums/eu/adonetefx/thread/0a794d93-67d1-45de-88dd-cab06267531b" rel="nofollow">Check this link</a></p>
