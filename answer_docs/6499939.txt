<p>If the input is valid XML (that is, if start and end tags always match, ie. <code>&lt;p&gt;...&lt;/p&gt;</code> or <code>&lt;P&gt; ... &lt;/p&gt;</code> but never <code>&lt;p&gt;...&lt;/P&gt;</code>) then you can process the data using XML tools such as XSLT. If the data isn't well-formed XML then you will need to use non-XML tools, e.g. Perl scripts. If the data has HTML-like characteristics then a tool like JTidy or TagSoup might turn it into something useful.</p>
