<p>My opinion is that you shouldn't modify this behaviour, since the fling behaviour is what the user expects.</p>

<p>However, to your question. I haven't tried this but in theory it should work.</p>

<p>Implement an <a href="http://developer.android.com/reference/android/widget/AbsListView.OnScrollListener.html" rel="nofollow"><code>OnScrollListener</code></a> to your <code>ListView</code> and use the <a href="http://developer.android.com/reference/android/widget/AbsListView.OnScrollListener.html#onScrollStateChanged%28android.widget.AbsListView,%20int%29" rel="nofollow"><code>onScrollStateChanged()</code></a> method to check if the current state is <a href="http://developer.android.com/reference/android/widget/AbsListView.OnScrollListener.html#SCROLL_STATE_FLING" rel="nofollow"><code>SCROLL_STATE_FLING</code></a>. After you've determined that the scrolling perfoms by a fling you can get your <code>ListView</code>'s first visible position by using the <a href="http://developer.android.com/reference/android/widget/AdapterView.html#getFirstVisiblePosition%28%29" rel="nofollow"><code>getFirstVisiblePosition()</code></a> method and from there you can use <a href="http://developer.android.com/reference/android/widget/ListView.html#smoothScrollToPosition%28int%29" rel="nofollow"><code>smoothScrollToPosition()</code></a> where you put in your <code>getFirstVisiblePosition()</code> value as an argument.</p>
